BUILD A COMMAND LINE EXPENSE TRACKER USING PYTHON PROGRAM

Project Description
Create an Expense Tracker that allows users to 
	-record daily expenses, 
	-view expenses, 
	-search, 
	-update, 
	-delete and 
	-generate a spending summary.
Store all expense data in a text file.

Expense Information
Each expense should contain:
	-Expense ID
	-Date (YYYY-MM-DD)
	-Category
	-Description
	-Amount

Example:
	1,2026-07-11,Food,Lunch at restaurant,350
	2,2026-07-11,Transport,Rickshaw,80
	3,2026-07-12,Shopping,T-Shirt,1200
Store all data inside: expenses.txt

Menu
========= Expense Tracker =========
1. Add Expense
2. View All Expenses
3. Search Expense
4. Update Expense
5. Delete Expense
6. Expense Summary
7. Exit

1. Add Expense
Ask the user for:
	Expense ID:
	Date (YYYY-MM-DD):
	Category:
	Description:
	Amount:

Create the function: def add_expense():

2. View All Expenses
Display all expenses in a readable format.
Example:
------------------------------------
	Expense ID : 1
	Date : 2026-07-11
	Category : Food
	Description: Lunch
	Amount : 350
------------------------------------
Create the function: def view_expenses():

3. Search Expense
Search by Expense ID.
If found:
Expense Found
	Expense ID : 2
	Date : 2026-07-11
	Category : Transport
	Description: Rickshaw
	Amount : 80
Otherwise:
	Expense not found.

Create the function: def search_expense():

4. Update Expense
Search using Expense ID.
Allow the user to update:
	-Date
	-Category
	-Description
	-Amount
Create the function:
def update_expense():

5. Delete Expense
Delete an expense using Expense ID.
Create the function: def delete_expense():

6. Expense Summary
Display:
	Total number of expenses
	Total amount spent
	Average expense
	Highest expense
	Lowest expense

Example:
========= Expense Summary =========
Total Expenses 	: 8
Total Spending 	: 7,850 BDT
Average Expense : 981.25 BDT
Highest Expense : 2,500 BDT

Create the function: def expense_summary():

7. Exit
Exit the program safely.


Exception Handling :
Your program must handle the following exceptions.
	Invalid Amount
		If the user enters: abc
		Output:
		Invalid amount! Please enter a valid number.
	Negative Amount : Amount cannot be negative.
	File Not Found 	: No expense records found.
	Empty File	: No expenses available.
	Invalid Menu Option : Invalid choice! Please select a valid option.
